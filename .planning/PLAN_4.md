# Phase 4: Diagram Generation

## Goal
实现将代码依赖的 JSON 图谱转换为 Mermaid 文本并渲染为交互式 AntV G6 图，确保中文字符正确显示。

## Success Criteria
1. 接口 `GET /diagram/{project_id}` 返回有效的 Mermaid 文本，中文字符不乱码。
2. 前端使用 AntV G6 渲染相同的图谱，支持缩放、平移、节点点击。
3. 单元测试覆盖图谱转换逻辑，覆盖率 ≥ 80%。
4. 性能基准：生成 10k 行代码的图谱不超过 2 秒。

## Requirements (from ROADMAP & REQUIREMENTS)
- VIS-01: 将统一的 JSON 依赖图转换为 Mermaid 文本。
- VIS-02: 支持 AntV G6 渲染交互式思维导图。
- VIS-03: 正确显示中文字符，使用 Noto Sans SC / Source Han Sans 字体。
- DEP-01: Docker Compose 中后端服务提供 `/diagram` API。
- SEC-01: 所有密钥通过环境变量管理，不在代码中硬编码。

## Tasks
1. **设计 JSON→Mermaid 转换模块** (`backend/diagram/converter.py`).
2. **实现 AntV G6 前端组件** (`frontend/src/components/Diagram.vue`).
3. **编写单元测试** (`tests/backend/test_converter.py`).
4. **添加性能基准测试** (`tests/performance/test_diagram_perf.py`).
5. **更新 Dockerfile** 以暴露新端口，如有需要。
6. **文档更新**：在 `docs/diagram.md` 中说明使用方法。

## Risks & Mitigations
- **中文渲染问题**：使用 Noto Sans SC，确保前端 CSS 正确加载。
- **性能瓶颈**：使用流式写入避免一次性生成大字符串。

- Task 3-4: 1 day
- Task 5-6: 0.5 day

- Manual UI test: load diagram for sample project, verify Chinese labels.
